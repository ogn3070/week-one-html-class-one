Interactive HTML5: Forms and Page structure (semantics)

HTML Forms Basics

HTML forms: Are essential for collecting user input.

Common form elements include:

  • <input>: text, password, email, number, date, checkbox, radio, file, submit, reset, button.
  • <textarea>: Multi-line text input.
  • <select>: Dropdown menus, option + optgroup.
  • <button>: Submit or reset buttons.

button vs input type= Submit

input submit is simpler, limited design options, while button submit is more flexible, modern, customizable

Common attributes for form elements are:

  • required: Ensures the field must be filled out.
  • placeholder: Provides a hint to the user about what to enter.
  • value: Specifies the initial value of the input.
  • checked: Indicates if a checkbox or radio button is selected.
  • selected: Indicates if an option in a dropdown is selected.
  • disabled: Prevents user interaction with the input.
  • readonly: Makes the input non-editable.
  • maxlength: Sets a limit on the number of characters.
  • minlength: Sets a minimum number of characters.
  • pattern: Specifies a regex pattern for input validation.
  • name: Identifies the input for form submission.
  • id: Uniquely identifies the input element.
  • class: Assigns one or more class names to the input.
  • style: Applies inline CSS styles to the input.
  • autocomplete: Controls browser autocomplete behavior.
  • autofocus: Automatically focuses the input on page load.

Form Grouping & Accessibility are:

  • <fieldset>: Groups related form elements together, often with a legend.
  • <legend>: Provides a caption for the content of a <fieldset>.

Associating label with inputs (for + id) must have a corresponding value and it improves accessibility and usability.

Semantic HTML5 Layout (Page structure)

Semantic HTML5 elements provide meaning to the structure of a webpage, enhancing accessibility and SEO.

  • <header>: Represents introductory content or a set of navigational links.
  • <nav>: Defines a section containing navigation links.
  • <main>: Specifies the main content of the document, unique to the document.
  • <article>: Represents a self-contained piece of content that could be distributed independently.
  • <section>: Defines a thematic grouping of content, typically with a heading.
  • <aside>: Contains content indirectly related to the main content, like sidebars or callouts.
  • <footer>: Represents the footer for its nearest sectioning content or sectioning root element.
